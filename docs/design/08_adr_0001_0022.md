## 各 ADR のキーポイント(面接で語る用)

### ADR-0001 Monorepo

**トレードオフ**: 初期設定コスト vs 共通化による往復削減。ルート学習コスト・Vercel/Fly のビルド対象指定が発生するが、共通パッケージ変更 1 PR で両アプリ反映・CI キャッシュ短縮・monorepo 運用経験アピールで上回る。

### ADR-0004 Socket.IO vs 生 WebSocket

**トレードオフ**: +6KB バンドル vs 自動再接続・ルーム・Redis Adapter。リアルタイム SaaS の品質要件(再接続・部屋分離)を自作すると保守コスト増で不採用。

### ADR-0007 楽観ロック

**トレードオフ**: UI マージ実装コスト vs UX・観測性。悲観ロックは WebSocket 切断時の解放漏れリスクがあり、楽観は 409 率で衝突頻度を可視化可能。

### ADR-0010 RLS 二重防御

**トレードオフ**: マイグレーション運用(migrator/app 分離)+ JOIN ポリシーコスト vs 情報漏洩ゼロ保証。Knowlex は tenant cross-access が致命事故に直結するため、ORM バグを DB で止める価値が上回る。

### ADR-0011 ハイブリッド + Rerank

**トレードオフ**: クエリ +300〜500ms レイテンシ vs Context Precision 0.27pt 向上。SSE で体感遅延を軽減して採用。

### ADR-0013 Faithfulness

**トレードオフ**: 応答完了後 +1.5s API コール vs 幻覚検出。業務利用耐性のため採用、UI に「未検証」マーク表示で可視化。

### ADR-0017 公開順序

**意思決定基準**: 16 週 2 本完走のリスク。Week 10 時点で Boardly 完成 → 応募開始可能。最悪シナリオでも「完成した 1 本」がある安全設計。
