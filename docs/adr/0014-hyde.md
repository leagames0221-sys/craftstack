# ADR-0014: HyDE (hypothetical document embeddings)

- Status: **Accepted (planned)** — design-phase decision; **deferred for v0.5.2** per [ADR-0039](0039-knowlex-mvp-scope.md) MVP scope
- Date: 2026-04-22
- Tags: ai, retrieval

> **Implementation status (v0.5.2)**: not implemented. The MVP corpus and question distribution don't yet justify the +1 LLM call per query; the eval cron would surface the gain measurably before HyDE ships. The "+8 points on the golden 50 set" measurement below is a design target, not a current measurement.

## Context

Short user queries embed far from the dense prose they are asking about, which hurts vector recall.

## Decision

Ask Gemini Flash to draft a hypothetical answer to the user query, then embed that draft for retrieval. Past conversation turns feed into the draft so follow-up questions stay on-topic.

## Consequences

Positive:

- Measured +8 points on the golden 50 set for vague questions
- Preserves conversational context without ballooning the retrieval prompt

Negative:

- One extra LLM hop (~300ms, few hundred tokens) per query
- Specific queries gain little; a length heuristic skips HyDE when the query already looks precise

## Alternatives

- Query rewrite only: retained alongside HyDE, not a replacement
- Multi-query retrieval: rejected — doubles cost for marginal gain
